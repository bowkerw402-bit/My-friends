# Standards

Will's rules. When he states a preference about how something should be done, it gets written here with
the date it was set. Both Claude and Codex read this folder before producing anything.

This exists because rules were previously scattered: some in Claude's private memory (which Codex cannot
read), some buried in chat. Now there is one shared place, and it grows.

## The files
| File | Covers |
|---|---|
| `writing-style.md` | how anything written for Will or on his behalf should read |
| `ways-of-working.md` | how work is run, prioritised, and gated |
| `visual-design.md` | how anything a person looks at gets approached |
| `email.md` | how outreach and replies get written |

## How a rule gets added
1. Will says it (in passing counts, he should not have to file it).
2. It gets written here as its own dated entry, in his words where possible, with the reason.
3. Newest entries sit at the top of each file so recent thinking wins.
4. If a new rule contradicts an old one, the old one is struck through, not deleted, with the date it changed.

## Precedence
Recent beats old. Explicit beats inferred. If a standard here conflicts with something an agent assumes,
the standard wins.
