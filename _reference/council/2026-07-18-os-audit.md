# Council verdict — audit of the two-agent operating system (2026-07-18)

**Question:** find (A) optimizations and (B) bugs/deficits in the newly built OS. MINI council with real
substrate diversity: 2 Claude advisors (Executor, First-Principles) + **Codex** as the bug-hunter.

## 1. Disagreement map (the product)
- **Ceremony vs lean** *(values choice — Will's call).* First-Principles: a raw run emits a 6-file Run Record where
  4 files (brief/plan/output/review) are "skipped" stubs — the 7 stations live in prose, not the machine; "don't
  ship records where two-thirds are apologies." Executor + the current design: keep the scaffolding. → **Resolve by
  weighting:** if you'll actually write a Brief before big runs → keep the stations (make FRAME/REVIEW real: brief-in,
  verdict-out). If not → drop to a lean record (transcript + tools + a 1-line SUMMARY) and make The Line an *optional*
  overlay for big work only.
- **Keep vs drop BOARD.md** *(minor, values).* First-Principles: it's a regrouping of `LEDGER.md`, a view-over-a-view
  for ~20 ventures — read the ledger. Recommendation: **keep** (it's cheap and the grouped glance earns its keep), but
  this is low-stakes.
- **Tool registry** *(resolvable by evidence).* First-Principles: retire it until connectors work (only 1/7 usable).
  Now partly mooted — the ledger is **verified** (real Claude tools captured), so the registry has a live purpose;
  reframe it as "what's authorized" + the verified ledger.
- **PII: regex vs rule** *(resolved, belt-and-suspenders).* First-Principles: a boundary rule ("vault never holds
  client dirs") is simpler/safer than a heuristic. Codex + Will: catch content too. → **Did both** — path boundary AND
  a conservative content scan that errs toward over-redaction (safe). Accepts a few false-positive redirects.
- **More (Executor) vs less (First-Principles).** Resolved: added the *ease* features that cut friction without adding
  conceptual weight (verbs), flagged the *cut* items as Will's direction call.

## 2. Convergences
- Both non-code advisors and Codex independently agreed: **default to requiring both agents** (silent single-agent is
  the failure mode). Done — `run` defaults to `-RequireBothAgents`.
- The system's core (honest two-agent accounting, draft-only, archive-never-delete) is sound; the issues are at the edges.

## 3. Blind spots (what ALL of them missed)
- **Cost / rate-limits.** Each run spawns Claude sub-instances; the CLI reported `out_of_credits` for overage. Nobody
  flagged per-run cost or rate-limit exhaustion as a scaling risk. Real operational ceiling.
- **Orchestrated agents can write.** They run with `--dangerously-skip-permissions`; in the validation run Claude
  ignored "READ-ONLY" and created `QUICKSTART.md`. Agents have write access to the work dir regardless of the task text.

## 4. Recommendation
Ship the fixes (done — see appendix). Take the **lean-by-default** path: orchestrate is a logged, health-checked
pair-run with a lean Run Record; The Line's Brief/Review are an *optional overlay* invoked for big work, not a tax on
every run. Keep BOARD. Treat the verified ledger as the registry's justification. Watch cost.

## 5. Tripwire + flip-evidence
- **Tripwire:** 2 weeks with no Brief written before a run → FRAME is theater; cut it to the lean record.
- **Flip-evidence:** any run corrupting another's record despite the fixes → the concurrency model needs a real queue, not just unique ids + a mutex.

## 6. First step
Decide the ceremony question (stations real vs lean). Everything else is already implemented.
