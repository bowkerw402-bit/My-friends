## Problems hit
- Codex was blind to everything past line 1 of the prompt, producing shallow "what are we building?" replies.
- The two-agent health check measured whether an agent replied, not whether it did the task, so a whiffing Claude still got labelled `two-agent`.
- Hero pixel breakup: `antialias: true` is inert once EffectComposer is active; water normal map had no anisotropic filtering.
- Supersampling cap was absolute (`min(max(devicePixelRatio, 1.6), 2)`), so on a 2x display it returned native resolution and supersampled nothing.
- Headless Codex CLI (v0.142.4) pinned to model `gpt-5.6-sol`, which the API rejects, so image regeneration cannot be auto dispatched.
- Deck was built as Will's teleprompter with stage directions printed on client facing slides (slide 7).

- File locks from OneDrive sync repeatedly blocked pptx/xlsx writes.

## Solutions found
- Codex now gets the full prompt via a file, not a mangled CLI argument. Proven: Codex read the code and returned five specific correct defects.
- Health check now measures task completion, not mere presence.
- Real SSAA via supersampling through the composer, plus anisotropic filtering on the water; cap made relative to display; FXAA added.
- Atomic export script `tools/export-site.sh` scrapes every route and verifies every referenced asset before declaring success.
- Settle driven handoff: the 3D model fires a "settled" event on the exact landing frame that triggers the logo fade, removing timer race.
- Deck rebuilt as sparse visual aids with all script living in speaker notes only.
- Lock handling made graceful (retry, filter temp lock files) rather than force closing files Will has open.
